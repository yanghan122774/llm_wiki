## 二、理解阶段

### 2.1 项目架构理解

调用 superpowers:brainstorming，用 3 个 Explore agent 并行探索：

1. **核心管线**：ingest.ts（117KB，两阶段 LLM：分析→生成）、page-merge.ts（三层合并保护）、frontmatter.ts（YAML 解析）
2. **项目结构**：Tauri 2 Rust 后端 49 个 command、React 19 前端、MCP Server 8 个工具
3. **类型系统**：GENERATION_WIKI_TYPES（9 个，给 LLM prompt 用）、WIKI_TYPE_DIRS（9 个，给路径推断用）

### 2.2 关键发现

- `GENERATION_WIKI_TYPES` 会注入 LLM 生成 prompt → Phase 1 **不能**往这里面加经验类型，否则会污染通用 ingest
- `WIKI_TYPE_DIRS` 纯前端路径推断用 → 可以安全追加
- `inferWikiTypeFromPath` 有正则回退机制 → 即使不加映射也能识别自定义目录，但加上映射更规范
- project.rs 的 dirs 数组在上一个会话已追加 6 个目录（bugs/decisions/howto/agent-errors/patterns/templates）

### 2.3 6 个经验目录的设计理解

用户澄清了设计意图：
- **前 7 个目录**（entities/concepts/sources/queries/comparisons/synthesis + findings/thesis/methodology）= 传统知识库，从 PDF/论文等**文档**提取概念
- **后 6 个目录**（bugs/decisions/howto/agent-errors/patterns/templates）= 经验系统，从 Claude Code 会话 **transcript** 中提取

两套平行体系，同一 wiki 项目中共存，靠 frontmatter 的 type 字段区分。

### 3.3 extract_experiences.py（`tools/extract_experiences.py`）

新建 ~280 行 Python 脚本，5 步流程：
1. 读取 JSONL transcript，过滤 thinking/attachment/system 消息
2. 按 16K 字符切分为 chunk
3. 每个 chunk 调用 LLM（经验提取 prompt，含已有页面列表做去重提示）
4. 解析 `===FILE ... ===END` 格式输出，按标题哈希去重
5. 写入 `raw/sources/experiences/YYYY-MM-DD-slug.md`

CLI 参数：`--transcript`（必需）、`--project`、`--domain`、`--output-dir`、`--dry-run`、`--api-base`、`--api-key`、`--model`

### 3.4 CLAUDE.md（`D:\work\llm_wiki\CLAUDE.md`）

新建项目级 Claude Code 指令文件，包含三类经验触发规则：

1. **会话启动预加载**：检测 `LLM_WIKI_PROJECT` → MCP 搜索 → 告知用户 top 5
2. **自动触发**：编译错误/运行时异常/配置问题/重复工作 → 主动搜索知识库 → 引用 [[PageName]]
3. **/exp 命令**：手动提炼当前会话经验 → 写入 wiki/ → 更新 index + log
