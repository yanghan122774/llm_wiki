# LLM Wiki 部署指南

**适用于:** 给团队成员部署 llm_wiki 应用

---

## 一、编译运行

编译完成后，还需要两步才能使用经验积累系统：

| 步骤 | 说明 | 文档 |
|------|------|------|
| 1. 创建 wiki 项目 | 在应用中新建项目，选 **Experience (🧠)** 模板 | 应用内操作 |
| 2. 连接 Claude Code | 配置 Hook / MCP / CLAUDE.md | 把 [agent-deploy-guide.md](agent-deploy-guide.md) 发给 Claude Code，让它自动配置 |

## 二、从源码编译

### 系统要求

| 要求 | 说明 |
|------|------|
| **Ubuntu ≥ 22.04** | Tauri v2 硬依赖 webkit2gtk 4.1 + libsoup 3.0，**20.04 及以下无法编译** |
| **Node.js ≥ 20** | 推荐 [nvm](https://github.com/nvm-sh/nvm) 安装 |
| **Rust** | 通过 rustup 安装 |
| **Python ≥ 3.10** | 经验提取脚本需要 |

> **⚠️ Ubuntu 20.04 用户：** Tauri v2 的 `webkit2gtk-sys` crate v2 只支持 webkit2gtk 4.1 API（soup3），而 20.04 只有 webkit2gtk 4.0 + libsoup 2.4。无法通过 `Cargo.toml` 修改或 feature flag 绕过。**必须先升级到 22.04+**。

### 获取源码

```bash
# 方式 1：Git clone
git clone https://github.com/yanghan122774/llm_wiki.git

# 方式 2：中国镜像（GitHub 不可用时）
git clone https://gh.llkk.cc/https://github.com/yanghan122774/llm_wiki.git

# 方式 3：Windows 打包拷贝到 Linux
# 在 Windows 上排除 node_modules/ 和 src-tauri/target/ 后压缩，到 Linux 解压
```

### 安装依赖

按顺序执行：

```bash
# 1. 系统编译工具链
sudo apt-get install -y build-essential

# 2. Rust（已安装则跳过）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source "$HOME/.cargo/env"

# 3. Node.js ≥ 20（已安装则跳过，推荐 nvm）
# curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
# nvm install 20

# 4. Tauri 2 系统依赖
sudo apt-get install -y \
  libwebkit2gtk-4.1-dev \
  libappindicator3-dev \
  librsvg2-dev \
  patchelf \
  protobuf-compiler \
  xdg-utils

# 5. Python 3
sudo apt-get install -y python3
```

### 编译运行

```bash
cd llm_wiki

# 主项目依赖
npm install

# MCP Server
cd mcp-server && npm install && npm run build && cd ..

# 启动开发模式
npm run tauri dev
```

首次编译 Rust 需要 **20-40 分钟**（下载编译 600+ crates），之后增量编译只需几秒。

### 编译中常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| `cargo: command not found` | Rust 未安装或未载入 PATH | `source "$HOME/.cargo/env"` |
| `linker 'cc' not found` | 缺少 C 编译器 | `sudo apt-get install -y build-essential` |
| `pkg-config: glib-2.0 >= 2.70` | glib 版本过低（Ubuntu 20.04） | 升级到 Ubuntu 22.04+ |
| `No package 'libsoup-3.0' found` | Ubuntu 20.04 无 soup3 | 升级到 Ubuntu 22.04+ |
| `No package 'webkit2gtk-4.1' found` | 未安装 webkit2gtk 开发包 | `sudo apt-get install -y libwebkit2gtk-4.1-dev` |

## 三、常见问题

**Q: 应用启动后窗口空白？**

先确认 API 后端是否正常（窗口空白但 API 正常也可使用）：
```bash
curl http://127.0.0.1:19828/api/v1/health
# 返回 {"ok":true,"status":"running"} 表示后端正常
```

如果 API 正常但窗口空白，通常是 VM GPU 虚拟化不完整导致 WebKitGTK 渲染失败。尝试：
```bash
# 强杀旧进程
sudo fuser -k 19827/tcp 19828/tcp 2>/dev/null
sleep 1

# 禁用 GPU 合成
WEBKIT_DISABLE_COMPOSITING_MODE=1 npm run tauri dev
# 或更激进的参数组合:
WEBKIT_DISABLE_COMPOSITING_MODE=1 WEBKIT_DISABLE_DISPLAY_COMPOSITOR=1 GDK_BACKEND=x11 npm run tauri dev
```

还不行的话，检查系统图形库：
```bash
dpkg -l | grep libwebkit2gtk
# 如缺失运行时库:
sudo apt-get install -y libwebkit2gtk-4.1-0 libgtk-3-0 libgdk-pixbuf2.0-0
```

**Q: 启动后端口冲突（Address already in use）？**

旧进程未完全退出，端口被残留占用：
```bash
# 查看占用者
ss -tlnp | grep -E '19827|19828'

# 强制释放端口
sudo fuser -k 19827/tcp 19828/tcp 2>/dev/null
sleep 1

# 重新启动
npm run tauri dev
```

**Q: Ubuntu 20.04 能装吗？**

**不能。** Tauri v2 硬依赖 `webkit2gtk 4.1` + `libsoup 3.0`，这两个库在 20.04 上不存在（20.04 只有 webkit2gtk 4.0 + libsoup 2.4）。尝试过的方案：
- `Cargo.toml` feature flags → 无效（wry 默认启用 linux-ipc-protocol）
- 自编译 webkit2gtk → 依赖链过深（需编译 Glib ≥ 2.70），不可行

**唯一解：升级到 Ubuntu 22.04+。**

**Q: 直接运行二进制文件报 "Connection refused"？**

```bash
# ❌ 错误做法：直接运行 target/debug/llm-wiki
./src-tauri/target/debug/llm-wiki
# → 页面显示 "Could not connect to localhost: Connection refused"
```

**根因：** Debug 版本的 Tauri 二进制**不内嵌前端**，它期望从 Vite 开发服务器（`localhost:1420`）加载页面。直接运行二进制时 Vite 没有启动。

```bash
# ✅ 正确做法 1：开发模式（同时启动 Vite + Tauri）
npm run tauri dev

# ✅ 正确做法 2：构建发布版再运行（前端已内嵌）
npm run tauri build
./src-tauri/target/release/llm-wiki
```

> `tauri dev` 读取 `tauri.conf.json` 中的 `devUrl` + `beforeDevCommand`，自动启动 Vite。

**Q: Health check 返回 `authRequired: true` / MCP 不可用？**

```bash
curl http://127.0.0.1:19828/api/v1/health
# 返回 {"authConfigured":false, "authRequired":true, "mcpEnabled":false}
```

**根因：** API 配置文件 `app-state.json` 中缺少 `apiConfig` 节点。API 服务从这里读取：
- `token` → Bearer 认证令牌
- `mcpEnabled` → 是否开放 MCP 搜索端点
- `enabled` → API 总开关

**解决：** 在 llm_wiki 应用窗口的 Settings → API Configuration 中填写，或直接编辑 `app-state.json`：

**Linux:** `~/.local/share/com.llmwiki.app/app-state.json`
**Windows:** `%APPDATA%/com.llmwiki.app/app-state.json`

确保文件中包含：
```json
{
  "apiConfig": {
    "token": "<你的认证令牌>",
    "mcpEnabled": true,
    "enabled": true
  }
}
```

改完后重启 llm_wiki 应用。

> 如果窗口空白无法操作，可以用 `curl` 直接调 API 编辑。

**Q: CLAUDE.md 里写了 MCP 工具（llm_wiki_search 等），但会话中不可用？**

**诊断流程：**

```bash
# 1. API 是否在线？
curl http://127.0.0.1:19828/api/v1/health
# 预期: {"ok":true, "status":"running"}

# 2. MCP 是否启用？
curl http://127.0.0.1:19828/api/v1/health | grep mcpEnabled
# 预期: "mcpEnabled":true

# 3. MCP Server 连接是否正常？
claude mcp list
```

**常见原因和修复：**

| 现象 | 原因 | 修复 |
|------|------|------|
| API 不在线 | llm_wiki 应用没启动 | 启动应用 |
| `mcpEnabled: false` | API 配置未开启 MCP | 参考上一条 FAQ，编辑 `app-state.json` |
| API 在线但 MCP 仍不可用 | MCP Server 注册后未重启 Claude Code | 重启 Claude Code 会话 |
| `claude mcp list` 中无 llm-wiki | 未注册 MCP Server | 把 [agent-deploy-guide.md](agent-deploy-guide.md) 发给 Claude Code，它会自动注册 |

> **核心概念：** `CLAUDE.md` → 告诉 Claude 用什么工具；MCP Server (`mcp-server/dist/src/index.js`) → 连接 Claude Code 和 API；`llm_wiki` 应用 → 提供 API（端口 19828）。三者缺一不可。

**Q: 如何卸载？**

删除 `llm_wiki` 源码目录即可。如果编译过发布版（`npm run tauri build`），还要删除 release 二进制和 `app-state.json`：
```bash
# Linux
rm -rf ~/桌面/llm_wiki
rm -f ~/.local/share/com.llmwiki.app/app-state.json

# Windows
# 删除源码目录 + %APPDATA%/com.llmwiki.app/
```

## 四、Web 端 / 无 GUI 模式

如果你不需要桌面窗口（比如在 VM 里 GPU 渲染有问题），可以只用浏览器访问前端，或者纯 API 模式。

### 开发模式（源码编译）

`npm run tauri dev` 启动时，Vite 开发服务器在 **`http://localhost:1420`**，可以直接在浏览器打开：

```
┌──────────────────────────────────────┐
│  npm run tauri dev                   │
│                                      │
│  ├─ Vite 前端  → http://localhost:1420  ← 浏览器访问
│  ├─ API 后端   → http://localhost:19828
│  └─ Tauri 窗口 → 可以最小化/忽略        │
└──────────────────────────────────────┘
```

Tauri 窗口只是一个 WebView 壳，和浏览器打开 `:1420` 看到的是**同一个页面**。VM 里窗口可能白板，但浏览器完全正常。

### 纯 API 模式

对于 Claude Code 经验系统，**只要能访问 `:19828` 的 API 就够了**——MCP Server 是通过 API 通信的，不依赖 GUI 窗口。启动应用后，所有操作可通过 API 完成：

```bash
npm run tauri dev &
curl http://127.0.0.1:19828/api/v1/health        # 健康检查
curl http://127.0.0.1:19828/api/v1/projects       # 列出项目
curl http://127.0.0.1:19828/api/v1/search?q=xxx   # 搜索
```

### 纯服务器模式（无桌面环境）

在没有 GUI 的服务器上，Tauri 需要图形环境才能启动窗口。变通方案：

```bash
# 安装虚拟显示
sudo apt-get install -y xvfb

# 在虚拟显示中启动
xvfb-run npm run tauri dev
# 或者
xvfb-run ./src-tauri/target/release/llm-wiki
```

> **当前限制：** Tauri 2 的 Rust 后端和 GUI 进程是同一个二进制，无法完全分离为独立的 headless 服务。后续版本计划支持独立 headless 模式。
