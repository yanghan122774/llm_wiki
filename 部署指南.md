# llm_wiki 完整部署指南

**日期:** 2026-06-24
**适用于:** 全新部署 llm_wiki 应用 + 经验积累系统

---

## 部署总览

```
安装系统依赖 → 安装语言运行时 → 克隆项目 → 编译运行 → 创建 wiki 项目 → 配置经验系统 → 验证
```

预计时间：首次 40-60 分钟（含 Rust 编译），之后日常启动只需 `npm run tauri dev`。

---

## 一、Windows 部署

### 1.1 系统依赖

| 组件 | 安装方式 | 用途 |
|------|---------|------|
| Node.js ≥20 | [nodejs.org](https://nodejs.org/) 下载安装 | 前端 + MCP Server |
| Rust | `winget install Rustlang.Rustup` 或 [rustup.rs](https://rustup.rs/) | Tauri 后端编译 |
| Python ≥3.10 | [python.org](https://python.org/) 下载安装 | 经验提取脚本 |
| protoc | `winget install protobuf` 或[手动下载](https://github.com/protocolbuffers/protobuf/releases) | Rust lance-encoding 编译需要 |
| Visual Studio Build Tools | 安装时勾选"使用 C++ 的桌面开发" | Windows 上 Rust 的 C 链接器 |

### 1.2 克隆项目

```powershell
git clone git@github.com:yanghan122774/llm_wiki.git
cd llm_wiki
```

### 1.3 编译运行

```powershell
# 安装前端依赖（首次 3-5 分钟）
npm install

# 安装 MCP Server 依赖
cd mcp-server
npm install
npm run build
cd ..

# 启动应用（首次编译 Rust 10-30 分钟）
npm run tauri dev
```

### 1.4 创建 wiki 项目

应用启动后，在窗口中操作：

1. 点击左侧 "New Project" 或 "+" 按钮
2. 项目名填入 `项目名-experiences`（如 `smart-lock-experiences`）
3. 模板选择 **Experience (🧠)**
4. 选择一个合适的父目录
5. 点击 Create
6. 记下项目路径（如 `D:\work\smart-lock-experiences`）

### 1.5 配置经验系统

> **推荐方式：** 把 [agent-deploy-guide.md](agent-deploy-guide.md) 发给 Claude Code，让它自动引导配置（覆盖已安装应用和源码两种模式）。如果 llm_wiki 已经在运行，也可以用 [agent-setup-guide.md](agent-setup-guide.md)。

**手动方式：**

1. 在你的开发项目根目录（写代码的项目）创建目录：
   ```powershell
   mkdir .claude\skills
   ```

2. 复制配置文件：
   ```powershell
   copy experience-system\config\settings.json <你的开发项目>\.claude\settings.json
   copy experience-system\skills\exp.md <你的开发项目>\.claude\skills\exp.md
   ```

3. 编辑 `.claude\settings.json`，替换占位符：

   | 占位符 | 替换为 |
   |--------|--------|
   | `<你的项目名>` | 开发项目名，如 `smart-lock-firmware` |
   | `<技术领域>` | 如 `embedded-arm` |
   | `<llm_wiki 安装目录>` | `D:/work/llm_wiki`（注意正斜杠） |
   | `<experience-system>` | `D:/work/llm_wiki/experience-system` |
   | `<你的wiki项目>` | 1.4 创建的 wiki 项目路径 |

4. settings.json 完整结构（嵌套格式）：
   ```json
   {
     "hooks": {
       "SessionEnd": [{
         "matcher": "",
         "hooks": [{
           "type": "command",
           "command": "python <experience-system>/tools/capture_session.py --stdin"
         }]
       }]
     },
     "env": {
       "LLM_WIKI_PROJECT": "<你的项目名>",
       "LLM_WIKI_DOMAIN": "<技术领域>",
       "LLM_WIKI_ROOT": "<llm_wiki 安装目录>",
       "LLM_WIKI_OUTPUT_DIR": "<你的wiki项目>/raw/sources/experiences"
     }
   }
   ```

5. 配置 MCP：

   - 检查 `C:\Users\<你的用户名>\.claude\.mcp.json` 是否存在
   - 不存在 → 复制 `experience-system\config\.mcp.json` 过去，把 `<llm_wiki安装目录>` 替换为 llm_wiki 的安装路径
   - 已存在 → 在 `mcpServers` 下追加：
     ```json
     "llm-wiki": {
       "command": "node",
       "args": ["D:/work/llm_wiki/mcp-server/dist/src/index.js"]
     }
     ```

### 1.6 验证

```powershell
# 1. 检查 MCP 连通性
curl http://127.0.0.1:19828/health

# 2. 在 Claude Code 中输入 xp 测试
```

## 二、Linux (Ubuntu/Debian) 部署

### 2.1 系统依赖

```bash
sudo apt-get update
sudo apt-get install -y \
  libwebkit2gtk-4.1-dev \
  librsvg2-dev \
  patchelf \
  protobuf-compiler \
  xdg-utils \
  libssl-dev \
  python3

# tray icon 依赖（可能冲突，失败不影响核心功能）
sudo apt-get install -y libayatana-appindicator3-dev 2>/dev/null || \
  echo "跳过 tray icon 依赖"
```

| 依赖 | 用途 |
|------|------|
| `libwebkit2gtk-4.1-dev` | Tauri WebView 运行时 |
| `librsvg2-dev` | SVG 图标渲染 |
| `patchelf` | 打包 AppImage 时修改动态链接 |
| `protobuf-compiler` | Rust lance-encoding 编译需要 |
| `xdg-utils` | AppImage 中 xdg-open |
| `libssl-dev` | Rust lancedb crate 需要 OpenSSL |
| `python3` | 经验提取脚本 |

### 2.2 语言运行时

```bash
# Node.js（需要 ≥20）
node --version   # 如果 <20，用 nvm 安装

# Rust（国内用镜像）
export RUSTUP_DIST_SERVER=https://mirrors.tuna.tsinghua.edu.cn/rustup
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env

# Python
python3 --version   # 应该已安装
```

> **注意：** Ubuntu 上 `python` 命令默认不存在，只有 `python3`。后续配置 settings.json 时 Hook command 要用 `python3` 而非 `python`。

### 2.3 克隆项目

```bash
mkdir -p ~/work && cd ~/work
git clone git@github.com:yanghan122774/llm_wiki.git
cd llm_wiki
```

### 2.4 编译运行

```bash
# 安装前端依赖（首次 3-5 分钟）
npm install

# 安装 MCP Server 依赖
cd mcp-server && npm install && npm run build && cd ..

# 启动应用（首次编译 Rust 20-40 分钟）
npm run tauri dev
```

### 2.5 创建 wiki 项目

同 Windows 1.4 — 应用窗口内操作，选 Experience 模板，记下路径（如 `/home/user/work/my-experiences`）。

### 2.6 配置经验系统

与 Windows 相同的步骤，但有以下几个 Linux 特定的注意点：

**① Hook 格式（嵌套结构，`matcher` + `hooks` 包装）用 `python3` 而不是 `python`：**

```json
{
  "hooks": {
    "SessionEnd": [{
      "matcher": "",
      "hooks": [{
        "type": "command",
        "command": "python3 /home/user/work/llm_wiki/experience-system/tools/capture_session.py --stdin"
      }]
    }]
  }
}
```

**② 所有路径用正斜杠（Linux 天然满足）：**

```json
{
  "env": {
    "LLM_WIKI_PROJECT": "smart-lock-firmware",
    "LLM_WIKI_DOMAIN": "embedded-arm",
    "LLM_WIKI_ROOT": "/home/user/work/llm_wiki",
    "LLM_WIKI_OUTPUT_DIR": "/home/user/work/my-experiences/raw/sources/experiences"
  }
}
```

**③ MCP 路径用 Linux 格式：**

```json
{
  "mcpServers": {
    "llm-wiki": {
      "command": "node",
      "args": ["/home/user/work/llm_wiki/mcp-server/dist/src/index.js"]
    }
  }
}
```

# 2. 检查 SessionEnd Hook 日志

## 三、ARM Linux（树莓派 / ARM64 服务器）

步骤与 x86_64 Linux 相同，但需要额外一步——替换 pdfium 为 ARM64 版本：

```bash
cd ~/work/llm_wiki
cp src-tauri/pdfium/libpdfium-arm64.so src-tauri/pdfium/libpdfium.so
file src-tauri/pdfium/libpdfium.so   # 确认是 aarch64
```

然后再执行 `npm run tauri dev`。

## 四、关键注意点

| 注意项 | Windows | Linux |
|--------|---------|-------|
| Python 命令 | `python` | `python3` |
| Hook command | `python ...` | `python3 ...` |
| 路径分隔符 | 正斜杠 `/` 或双反斜杠 `\\` | 正斜杠 `/` |
| 系统托盘依赖 | 无需额外操作 | `libayatana-appindicator3-dev`（可能冲突，跳过） |
| 透明标题栏 | 正常 | 部分窗口管理器可能有 UI 问题 |
| 文件监听 | 正常 | inotify 可能漏事件，代码有定期轮询补偿 |
| SSH 克隆 | 可能需要配 VPN（端口 22 被墙） | 同左 |

## 六、常见问题

### Q: `libappindicator3-dev` 安装失败？
跳过即可，不影响核心功能。仅影响系统托盘图标。

### Q: 首次 `npm run tauri dev` 很慢？
正常。首次要下载 Rust crates 并编译，可能 20-40 分钟。之后增量编译只需几秒到几十秒。

### Q: `protoc` 找不到？
```bash
# Linux
sudo apt-get install -y protobuf-compiler
# Windows
winget install protobuf
```

### Q: MCP 搜索无响应？
检查 llm_wiki 应用是否正在运行，端口 19828 是否被占用。

### Q: SessionEnd 后经验没入库？
1. 检查 `capture_session.log` 是否有报错
2. 检查 settings.json 中 hook command 的路径是否正确
3. 检查环境变量 `LLM_WIKI_OUTPUT_DIR` 指向的目录是否存在
4. Linux 上确认用的是 `python3` 而非 `python`

### Q: 如何卸载？
1. 从开发项目 `.claude/settings.json` 中删除 `hooks.SessionEnd` 和 `env` 段
2. 删除开发项目 `.claude/skillsxp.md`
3. 从开发项目 `CLAUDE.md` 中删除"经验系统"段落
4. 从 `~/.claude/.mcp.json` 中删除 `llm-wiki` 条目
5. 删除 llm_wiki 目录即可
